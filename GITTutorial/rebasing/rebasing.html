Rebasing

INTRODUCTION

Rebasing is the process of re-applying a current branches commits on top of another 'tip'. Normally done when the remote branch is 'ahead' of the local branch. Should be done to prepare for a commit.

Assume the following history exists and the current branch is "topic" 

          A---B---C topic
         /
    D---E---F---G master

From this point, the result of the following command:

git rebase master

would be:

                  A'--B'--C' topic
                 /
    D---E---F---G master

ACTION:

Rebase Branch
git rebase

If f <branch> exists (with the same name) in the remote, it will be 'linked' to that remote branch.

ACTION:

Rebase Branch
git rebase <branch>

If f <branch> exists (with the same name) in the remote, it will be 'linked' to that remote branch.

git rebase Performs a rebase on by moving the 'base' of the to the 'tip' of branch.

ACTION:

Create and Switch Branch (using an existing branch as a 'base')
git checkout -b <new_branch> <base_branch>

<base_branch> must exist locally. This is the transactional equivalent of:

$ git branch -f <new_branch> [<<start point>]  (1)
$ git checkout <new_branch>                    (2)

  1. create branch from another branch

  2. switch to newly created branch
