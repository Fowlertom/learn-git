This page focuses on the common 'processes / workflows' that need to be performed. The 'most frequently used' process(es) are listed first.

Commit (AKA 'Check-in') Code

The following details the normal (happy path) workflow.

Action Command
Switch to 'base_branch'git checkout <base_branch>
Get latest of 'base_branch'git pull
Create local 'feature_branch'git checkout -b <feature_branch> <base_branch>
*****Edit files / do some work*****-------------
Stage changes to 'feature_branch'git add *
Commit changes to 'feature_branch'git commit -m "yourcomments"
Push changes to remotegit push -u origin <feature_branch>
Initiate merge to 'base_branch'create pull request (via web browser)
Complete merge to 'base_branch'complete pull request (via web browser)
Switch to 'master' (or any other branch)git checkout <master>
Delete local 'feature_branch'git branch -d <feature_branch>

Delete (aka 'Prune') local orphaned branches

There may be 'pointers' that are orphaned when a Pull Request removes the local 'feature_branch'. The following details how to remove local orphan branches / pointers / references that do not exist in the remote

Action Command
Get list of orphaned branchesgit branch -a
Determine what branches will be pruned (but do not prune them)git remote prune origin --dry-run
Prune orphaned branchesgit remote prune origin
