Undo Changes

If you want to undo/undelete/restore/discard changes done to your branch, following the steps below. The steps to follow depend on the state of the changes.

I Want To:
  • Undelete Files / Folders - accidentally deleted
  • Undo Changes - undo changes made to files/folders. These are displayed in 'red' text
  • Unstage File - files displayed in 'green' text
  • Revert Committed Change - revert (rollback) a committ to a branch
  • Make Everything The Same - make local branch exactly match remote (if you don't know what to choose)

Note: This assumes the branch to 'undo' is the 'current' branch. Use git checkout <feature_branch>

Undelete Files / Folders

Choose this when you accidentally delete files or folders, but have not yet staged them.

Action Command
Undelete files / foldersgit restore [FILENAME]

Undo Changes

Choose this when you want to undo/revert changes made to files folders.

git status will display eligible files/folders in 'red' text.

Action Command
Undo (unstaged) changesgit restore .

Unstage File

Choose this when you want to remove files/folders from being commited.

git status will display eligible files/folders in 'green' text.

Action Command
Unstage Filegit reset [FILENAME]

Committed Changes

Action Command
Undo changes done to <feature_branch>git restore .

Everything

Action Command
Switch to <feature_branch>git checkout <feature_branch>
Undo changes done to <feature_branch>git reset --hard <Origin/feature_branch>
Delete any untracked files & foldersgit clean -d -f
