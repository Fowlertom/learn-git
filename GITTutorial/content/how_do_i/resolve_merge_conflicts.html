Resolve Merge Conflicts

If a <target_branch> is ahead of a <source_branch>, you may encounter conflicts. This page describes how to manually resolve merge conflicts.

Microsoft Visual Studio Code is the best tool (in my opinion) to resolve merge conflicts.

Resolve Conflicts Manually

Action Command
Start Visual Studio Code-------------
Navigate to the local git repositoryClick 'File | Open Folder...'
Open a terminal windowClick 'Terminal | New Terminal'
Switch to <source_branch>git checkout <source_branch>
Get latest of <source_branch>git pull
Switch to <target_branch>git checkout <target_branch>
Get latest of <target_branch>git pull
Merge commits of <source_branch> into <target_branch>git merge <source_branch>
If conflicts are reported, navigate to Soure ControlClick 'View | SCM'
Select file with conflict Under 'Merge Changes' select the file with a 'C' on the right hand side
Navigate to the conflict<<<<<<< HEAD
code from target_branch
=======
code from source_branch
>>>>>>> <source_branch>
Resolve the conflictAbove '<<<<<<< HEAD', click either:
  • 'Accept Current Change'
  • 'Accept Incoming Change'
Perform the above two steps until all conflicts have been resolved-------------
Save the file, close the fileClick 'File | Save', Click 'File | Close Editor'
'Stage' the changed fileClick the 'Plus' icon. The icon is visible when you hover over the file.

Note: the file may not appear in the 'staged changes' section if you have chosen the changes that are already in the <target_branch>
Conclude resolving conflictsgit merge --continue
Enter the commit message, then close commit window In the editor window, enter the commit message on the first line. Click 'File | Close Editor'
