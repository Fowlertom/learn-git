Push

DETAILS

Performing a 'push' is the process of updating a 'remote branch' with anything missing from a 'local branch'.

git push -u origin <feature_branch>

Updates the <origin/feature_branch> with the missing commits from <feature_branch> If <origin/feature_branch> does not exist, it is created. The '-u' command adds upstream tracking reference - in case 'git pull' without arguments is used.

EXAMPLE

Assume the following history exists:

          A---B---C origin/feature_branch
         /
    D---E---F---G feature_branch
        ^
        origin/feature_branch in your repository

From this point, the result of the following command:

git pull

would be:

          A---B---C origin/feature_branch
         /         \
    D---E---F---G---H feature_branch
