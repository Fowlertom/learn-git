Merge Branches (<source_branch> into <target_branch>)

These steps will merge the contents of <source_branch> into <target_branch> in your . There are two different ways to merge code:

  • Rebase

There is a great article by Atlassian on merging vs rebasing.

In summary, retains history while rebasing re-writes history. The importance of history will dictate the method chosen.

Merging

If the <target_branch> is NOT ahead of the <source_branch>, the easiest way to perform the merge is using the TFS Pull Request.

WARNING: Merging using a TFS Pull Request should only be done if the <target_branch> is NOT ahead of the <source_branch>.


If the <target_branch> is ahead of the <source_branch>, you will need to merge the branches locally and resolve any conflicts. The following steps describe how to perform a 'local' merge.

Action Command
Switch to <source_branch>git checkout <source_branch>
Get latest of <source_branch>git pull
Switch to <target_branch>git checkout <target_branch>
Get latest of <target_branch>git pull
Merge commits of <source_branch> into <target_branch>git merge <source_branch>
Resolve conflicts (if any exist)
Commit changes to <target_branch>git commit -m "yourcomments"
Push changes to remote (if <target_branch> does not existgit push -u origin <target_branch>
(Force) Push changes to remote (if <target_branch> exists)git push -f origin <target_branch>

Rebasing

Action Command
Switch to <source_branch>git checkout <source_branch>
Get latest of <source_branch>git pull
Switch to <target_branch>git checkout <target_branch>
Get latest of <target_branch>git pull
Merge commits of <source_branch> into <target_branch>git rebase <source_branch> <target_branch>
(Force) Push changes to remote <target_branch>git push -f origin <target_branch>
